1. Introducción
	- Preguntas retóricas.
	- El trabajo realizado consiste en el diseño e implementación de una 
		aplicación que permita crear y planificar rutas turísticas. 
	- En lo que al turismo se refiere... -> aplicación móvil.
	- Destinada a cualquier usuario que quiera planificar sus viajes.
	
2. Objetivos
	- Planificar Rutas -> objetivo primordial (fechas, tiempos, distancias....)
	- Acceso a datos externos -> API Foursquare y Google.
	- Gestionar Eventos y Usuarios -> ¿quitar?
	- App móvil y web -> Dos clientes
	- Datos en tiempo real -> Se pretende obtener los datos reales de ejecución
		de la ruta para ser comparados con la planificación.

3. Demo
	- Aplicación Móvil
		- Mencionar pantalla registro y login.
		- Explorar rutas (Aplicando filtros)
		
		- Crear ruta -> Autocompletar de google.
		
		- Panel de Ruta/Viaje:
			- Asignar rango fechas
			- Itinerario (Selector de días)
				- Añadir sitio
					- Lugares recomendados de FS
					- Ver en mapa (Función "filtrar aquí")
					- Añadir lugares a los diferentes días
					- Filtros FS -> Explorar permite consultar por las diferentes
						categorías que gestiona FS
				- Obtener distancias y duración (Clickar de nuevo permite alternar
					entre los diferentes métodos de viaje) (Tiempos y distancias
					obtenidos de la API de Google)
				- Asignar tiempo de para (Mencionar -> modificar las horas de los
					sitios, altera la planificación total del día)
				- Opciones - Habilitar edición -> Modificar orden
				- Opciones - Modificar Hora salida -> Mencioar hora de salida por
														defecto 09:00.
				- Opciones - Ver en Mapa -> Lista horizantal en la parte inferior
												con los lugares por orden
				- Eliminar una visita.
			- Mapa -> No se realiza, es lo mismo que ver día en mapa.
			- Eventos
				- Próximos y En Mi Viaje
				- Lista con los días en los que hay un evento.
				- Para cada evento -> lista con las actividades o ubicaciones que
					lo forman.
					- Ver evento en mapa -> Mostrar en mapa y comparar con la ruta
						planificada en ese día
					- Añadir evento -> Añade evento al día correspondiente de la
						ruta
					- Eliminar evento -> Elimina el evento de la ruta.

			- Track -> No disponible, la ruta aun no ha comenzado

				- Opción de hacer privada la ruta.

			- Seleccionar Ruta en Curso.
				- Función track disponible.
				- Ver en mapa para mostrar una simulación de la ruta en tiempo
					real.
			- Opciones - Modificar Datos Personales 
				- Permite modificar el correo y la contraseña del usuario.
				- Salir -> te desconecta de la aplicación.

5. Análisis de Alternativas.

	- FS no vale para rutas.

6. Tecnologías de Desarrollo
	- Proyecto Java con Maven donde se ha utilizado Spring, JAX-RS...
	- Cliente web elaborado dentro del proyecto Java utilizando Spring MVC,
		Thymeleaf como motor de plantillas y tecnologías web como JS, AJAX, Boots...
	- Cliente móvil elaborado con el SDK Ionic, mediante el uso también de 
		tecnologías web.
	- Oracle ha sido el gestor de base de datos y se ha utilizado tomcat como
		servidor de aplicaciones.
